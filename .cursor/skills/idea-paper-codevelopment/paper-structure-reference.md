# idea/paper.md 结构对照

与仓库 `idea/paper.md` 示例对齐时的注意点（撰写终稿时自查）。

| 区块 | 要点 |
|------|------|
| 标题 | 准确概括方法 + 场景或对象 |
| 摘要 | 背景、痛点、方法名/要点、实验结论量化或趋势、意义 |
| 关键词 | 方法术语、任务域、应用场景 |
| §1 引言 | 1.1 分点陈述局限（可加粗小标题）；1.2 编号贡献，与后文章节呼应 |
| §2 相关工作 | 每小节先述经典定义或公式，再落到「与本文关系」 |
| §3 方法论 | 符号表、问题形式化、分小节算法；必要时 `$$` 块公式 |
| §4–§5 | 示例中为可解释性、信息熵分析；可按论文主题替换，保持论证深度 |
| §6 实验 | 设置表、主结果表、鲁棒性/消融；可含「建议在论文中补充的图」类备注 |
| §7 结论 | 编号条列，回应贡献，一句展望或应用边界 |
| 参考文献 | GB/T 7714 风格或期刊惯例；与正文引用顺序一致 |

数学与排版：行内 `$...$`，独立公式 `$$...$$`；表格用 Markdown 管道表。
